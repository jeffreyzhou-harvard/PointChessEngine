# Debate Evals

Debate candidates should be evaluated on both final output and debate process quality.

## Metrics

- tests passed
- review score
- bugs caught before implementation
- interface compatibility
- cost
- latency
- number of critique issues resolved
- final benchmark impact

## Debate-Specific Report Fields

Candidate reports for debate systems should include:

- model assignments by role
- whether the debate was homogeneous or heterogeneous
- number of rounds completed
- critique issues raised
- critique issues resolved before implementation
- final unresolved concerns
- tests/evals outcome

## Promotion Policy

Debate candidates are promoted only through the same Champion-mode gate as every other candidate. Debate quality can improve review and documentation scores, but it cannot compensate for failed contract tests.
