# Debate Roles

## Architect

Designs the implementation shape, module boundaries, and integration approach.

## Chess Correctness Expert

Focuses on rules, legality, FEN, perft, UCI move notation, and edge cases.

## Search/Performance Expert

Focuses on evaluation/search complexity, time controls, transposition tables, and benchmark impact.

## Skeptic

Challenges assumptions, looks for hidden failures, and identifies missing tests.

## Builder

Implements the selected plan in the candidate branch.

## Reviewer

Reviews the patch using the code review rubric and interface safety rules.

## Judge

Selects or synthesizes the plan before implementation. The judge does not override test/eval results.
